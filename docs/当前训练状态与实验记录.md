## 2. 模型架构

```
输入
├── img_clip     [B, 3, 448, 448]   → CLIP RN50x64 (全程冻结)      → clip_feat  [B,1024]
├── img_highres  [B, 3, 512, 512]   → SRMConv + ConvNeXt-tiny      → tex_feat   [B,768]
│                                                                   → skip_64   [B,192]
│                                                                   → skip_128  [B,96]
├── lare_map     [B, 7, 32, 32]     ─┐
└── luma_map     [B, 1, 32, 32]     ─┴→ SpatialLuminanceGate + ChannelInteraction
                                             ↓
               ┌─────────────────────────────┴──────────────────────────────┐
               │ 分类路                                                      │ 定位路 (独立权重)
               │ lare_conv → lare_feat [B,64]                               │ loc_stem → CBAM×3
               │ fusion_mlp(1856→512→128→2)                                 │ loc_head → coarse [B,1,32,32]
               │ → logits [B,2]                                             │ loc_refine64 + skip_64
               │                                                            │ → mid_head → mid [B,1,64,64]
               │                                                            │ loc_refine128 + skip_128
               │                                                            │ → fine_head → fine [B,1,128,128]
               └────────────────────────────────────────────────────────────┘
```

**关键设计决策：**
- 分类路与定位路使用**独立的 luma_gate/ch_interact 权重**，防止 seg_loss 梯度污染分类骨干
- `LocalizationRefineBlock` 带残差连接（shortcut），梯度稳定
- `mid_head` / `fine_head` 初始化为全零，旧 checkpoint 热启动时默认退化为上采样粗图

## 5. 定位退化根因分析

### Phase 2 失败（v14基线 → 57.9% Dice）

| 原因 | 影响 |
|------|------|
| `heavy_geo_transform` 空间错位 | highres+mask 被 ShiftScaleRotate/ElasticTransform 变形，但 loss_map(7ch,32×32) 未同步变形 → 模型在错误位置检测篡改 |
| 解冻过多层（luma_gate/ch_interact/loc_stem） | 破坏了 v14_fused_ready.pth 的已训练特征预处理权重 |
| boundary_weight=3.0 | 强迫拟合注释边界噪声 → 过预测 |

**修复：** 禁用 heavy_geo_transform（`self.heavy_geo_transform = None`），FLHLoss 去掉 boundary 逻辑，unfreeze_names 恢复最小集。

### V2/V3 定位退步（53-54% Dice）

**根因：定位监督比例被稀释**

```
v14基线 (doubao×15):  有mask样本占 ~85%
v2_balanced:          有mask样本占 ~41%
v3_full:              有mask样本占 ~25%  ← 更差
```

注意：无mask的SDXL/Flux假图**已被 `has_mask` 逻辑排除**在 seg_loss 之外，不是直接污染，而是骨干网络的梯度拔河失衡——75%的反向传播只有分类梯度在更新骨干，定位所需的空间特征被逐渐压制。

**当前修复方案：** `SEG_LOSS_SCALE=3.0`，放大定位梯度贡献，重建平衡。

## 6. 完整训练流程

```powershell
# Step 1: 重新生成注释（数据有变动时）
python script\gen_annotations_v2.py

# Step 2: 构建提取列表
python script\3_build_extract_list.py
# 输出: annotation/extract_v2.txt

# Step 3: 增量提取 SSFR 特征（已有 .pt 自动跳过）
python script\2_extract_features.py `
    --input_path annotation\extract_v2.txt `
    --output_path dift.pt `
    --extractor_type ssfr `
    --unet_path outputs\ssfr_unet.pth `
    --bf16 --batch_size 4 --num_workers 0

# Step 4: 训练
python script\5_train_model_v11.py `
    --resume outputs\v14_fused_ready.pth `
    --out_dir outputs\<实验名>

# Step 5: 生成大测试集（首次或数据变动后）
python script\gen_test_set.py

# Step 6: 分类评估
python test\evaluate_by_category.py `
    --model outputs\<实验名>\best.pth `
    --test_file annotation\test_v2.txt

# Step 7: 定位评估
cd test
python evaluate_localization_phase1.py `
    --model ..\outputs\<实验名>\best.pth `
    --ann_file ..\annotation\eval_change.txt `
    --map_file ..\dift.pt\ann.txt `
    --data_root ..\data
cd ..
```
