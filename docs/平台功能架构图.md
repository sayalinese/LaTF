## 4. AI推理数据流

```mermaid
sequenceDiagram
    participant U as 用户浏览器
    participant F as Flask Backend
    participant LM as LaFTManager
    participant LE as LareExtractor
    participant TF as TruForExtractor
    participant V11 as LaREDeepFakeV11
    participant CI as CascadeInference

    U->>F: POST /predict (multipart/form-data)
    F->>F: load_uploaded_image() → PIL RGB
    F->>LM: predict(img)
    LM->>LE: extract_single(img) → LaRE重建误差图
    Note over LE: SDXL扩散模型推理<br/>耗时约2-5s
    LM->>TF: extract_batch([img]) → 篡改可靠性图
    Note over TF: TruFor模型推理
    LM->>V11: forward(clip_feat, texture_feat, lare_feat)
    Note over V11: CLIP + ConvNeXt + LaRE<br/>三流融合分类
    V11-->>CI: 全局logits + 特征图
    CI->>CI: 全局置信度判断
    alt 置信度 > CASCADE_THRESHOLD(0.3)
        CI->>CI: 提取关注区域 → 裁剪放大
        CI->>V11: 局部区域二次推理
        CI->>CI: 融合全局+局部概率
    end
    CI-->>F: {prediction, confidence, heatmap, loc_map}
    F->>F: refine_map_for_visibility() → Base64热力图
    F-->>U: JSON响应 {class_name, confidence, heatmap}
```

## 5. VLM多模态推理流

```mermaid
sequenceDiagram
    participant U as 用户浏览器
    participant FE as Vue前端
    participant F as Flask Backend
    participant VLM as Qwen2.5-VL-72B

    U->>FE: 点击"生成VL报告"
    FE->>FE: fetch(imgUrl) → Blob
    FE->>FE: FileReader → Base64
    FE->>F: POST /explain_logic_stream\n{image_base64, is_fake}
    F->>F: 构造系统提示词\n(告知底层检测结果)
    F->>F: 构造用户提示词\n(要求视觉证据分析)
    F->>VLM: 图像+文本多模态请求\n(stream=True)
    loop SSE流式返回
        VLM-->>F: token chunk
        F-->>FE: data: {chunk}\n\n
        FE->>FE: vlReport += chunk
    end
    FE->>U: Markdown实时渲染
    Note over FE: 支持<think>标签\n推理链折叠展示
```

## 6. 会话纠纷解决流程

```mermaid
flowchart TD
    A[买家创建会话\nPOST /sessions] --> B[生成邀请码/链接]
    B --> C[卖家通过邀请码加入\nPOST /sessions/id/join]
    C --> D[双方在会话中发送消息]
    D --> E{消息类型}
    E -->|文本| F[普通文字消息]
    E -->|图片| G[上传图片消息\nPOST /sessions/id/messages]
    G --> H{触发AI检测?}
    H -->|手动点击检测| I[POST /sessions/id/detect_message]
    I --> J[Flask调用LaFTManager.predict]
    J --> K[返回检测结果写入\nai_detect_data字段]
    K --> L[前端展示检测结果卡片]
    L --> M{需要深度分析?}
    M -->|点击生成VL报告| N[startVlStream 流式VL分析]
    N --> O[Qwen-VL生成Markdown报告]
    O --> P[AI助手给出判决意见]
    P --> Q[会话状态更新\nopen → closed]
```

## 8. 关键配置参数

| 参数 | 值 | 说明 |
|------|----|------|
| `AI_CONFIDENCE_THRESHOLD` | `0.5` | 最终真/假判断阈值 |
| `CASCADE_THRESHOLD` | `0.3` | 触发局部级联检测阈值 |
| `CROP_SIZE` | `224` | 局部裁剪尺寸 |
| `INPUT_SIZE` | `448` | 全局输入尺寸 |
| VLM 提供商 | SiliconFlow / Ollama | `.env` 配置 |
| 数据库 | PostgreSQL | `DATABASE_URL` 环境变量 |
| 前端端口 | `:3000` | Vite Dev Server |
| 后端端口 | `:5000` | Flask |
